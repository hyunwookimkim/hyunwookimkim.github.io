spring boot

spring boot안에는 tomcat server와 springframwork가 내장되어 있다.
정적자원:실행하지 않고 그대로 읽음.tomcat server만 써도 동작이 가능(node.js가 이 방식임)
springframwork을 쓰는 이유는 읽어서 리턴하는게 아니라 실행해야 할때 사용함.

자바를 실행할때는 그냥 실행이 불가능하고 .java를 컴파일 해서 바이트 코드인 .class로 변환해서 사용한다.
컴파일 할때 쓰는 여러 도구들은 jdk(java development kit)라고 한다.
java로 만들어진것 들과 개발도구는 JVM(JAVA VIRTUAL MACHINE)이라는 인터프리터에서 실행한다.

자바는 하이브리드 방식이다.
하이브리드 방식을 도입한 이유:
1. os마다 빠로 컴파일을 할 이유가 없다.(동일한 바이트 코드를 생성)
2.os에 상관없이 인용 jvm이 설치 되어 있다면 실행할 수 있다.(운영체제에 종속되지 않음)
3.컴파일 과정에서 문법의 오류를 모두 찾아낸다.(실행할때마다 문법오류 검사를 해서 순수 인터프리터 보다 빠르다.)
4.완전기계어는 아니지만, 기계어와 가까운 언어로 변환된 명령을 실행하기 때문에 저장된 명령을 실행하는것 보다 빠르다.

LLVM 이전: 중복개발이 불편하고, 컴파일러 제작자가 여려언어와 여러기계를 다뤄야 하는 문제 생긴다.
같은 언어라도 OS가 다르면 각각 compile을 했다.
같은 os라도 언어가 다르면 각각 compile을 했다.

LLVM 이후: 새 프로그래밍 언어와 컴파일을 만들기 쉬워짐.
IR(중간언어)로 방식을 바꿔버렸다. 컴파일러 제작자는 한가지의 기계어(가상기계어)만 다루면 되는 장점이 있다.

=> 3단계로 쪼개면서 한번에 개발하는 것에 대한 부담이 줄었다.
기존에 만들어 놓은 컴파일러를 재사용 할 수 있다.

.kt .groovy .scala (코틀린, 그루비, 스칼라)는 자바의 호환언어로서 바이트코드로 컴파일 변환이 가능하고 JVM에서 실행이 가능하다.

.class를 실행시키기 위해서 JVM을 사용해야 하며 제일 대표적인 JVM은 JAVA.EXE이다.
.class를 사용하기 위해서는 java libray를 사용한다. 그리고 jvm은 실행시키기 위해서 같은 java libray를 필요로 한다.
그리고 JVM+java libray 를 JRE(자바를 실행시키기 위한 환경) 이라고 부른다.
또 그 JRE+컴파일나 디버거등의 개발도구를 필요로 하는데 그걸 JDK라고 한다.

JAVA EE는 기업용 application에 필요한 도구이다.(기술 명세서)
JAVA SE는 자바 어플리케이션 제작의 필수도구이다
(-JAVA EE를 다운받는다고 해서 JAVA SE가 다운 받아지지는 않는다.)
(-JAVA SE중에 server&JRE가 있는데, 기존 JRE에서 UI를 제거한 것이고, *서버전용도구 이다.) JAVA ME는 임베디드 애플리케이션 제작에 필요한 도구이다.(H/W에 삽입되는 APP)
